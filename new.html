Learn Git & GitHub

Git is a distributed version control system. GitHub is a platform for hosting Git repositories, collaborating on code, and managing projects.

What is Git?

Git helps you track changes to files over time, manage versions of your code, and collaborate with others. It works locally on your computer and stores a complete history of your project.

  • Version control system for source code
  • Tracks file changes and history
  • Supports branching and merging
  • Works offline and syncs with remote repositories

What is GitHub?

GitHub is an online service that hosts Git repositories and provides tools for collaboration, issue tracking, pull requests, and project management.

  • Remote repository hosting
  • Collaboration with pull requests and code review
  • Issue tracking and project boards
  • Continuous integration and community features

Basic Git Commands

Filter commands and copy the text in one click.

git init

Create a new Git repository in the current folder.

git clone

Copy an existing remote repository to your local machine.

git status

Show the current state of the working directory and staging area.

git add

Stage changes so they can be included in the next commit.

git commit

Save staged changes to the local repository history.

git push

Send committed changes from your local repository to a remote repository.

git pull

Fetch and merge changes from a remote repository into your local branch.

git branch

List, create, or delete branches in your repository.

How to Learn Git & GitHub

  1. Install Git and configure your username and email.
  2. Create a repository and initialize it with git init.
  3. Practice staging and committing changes with git add and git commit.
  4. Use git status often to understand what changed.
  5. Clone repositories from GitHub using git clone.
  6. Push your local commits to GitHub with git push.
  7. Collaborate using branches, pull requests, and code review.
  8. Use git pull to stay updated with remote changes.

Things to Learn

Repository Workflow

Understand the working directory, staging area, and commit history.

Branching

Learn how to create branches, switch branches, and merge code safely.

GitHub Collaboration

Use forks, pull requests, and reviews for team development.

Resolving Conflicts

Practice handling merge conflicts and using git diff.

Quick Reference

git init - Start a new local Git repo.

git clone <url> - Copy a repo from remote.

git add . - Stage all changed files.

git commit -m "message" - Record changes.

git push - Upload commits to GitHub.

git pull - Download and merge remote changes.
